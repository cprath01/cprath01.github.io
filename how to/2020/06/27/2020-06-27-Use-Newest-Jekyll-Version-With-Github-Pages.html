Use Jekyll 4.1.1 with Github Pages

So you want to take advantage of GitHub pages to build a website or blog. You notice that you can use Jekyll to build the site. You down load the latest version of Jekyll. Build the site and up load to a GitHub repo. And then â€¦ Nothing happens.

You probably ran into the same issue I did. Where at the time of this writing the latest version of Jekyll is 4.1.1 and according to GitHub Pages Dependency versions page the supported jekyll version is 3.8.7. So what do then.

You have a couple of options.

  1. Downgrade the local version of jekyll to the supported version.
  2. Not use Jekyll and build static pages from scratch.
  3. Use some other tech stack like React to render the static pages.
  4. Use the newest version of Jekyll and copy the _site folder into your GitHub Pages Repo

I did the fourth options.

The rest of the article assumes you have Git installed, a GitHub account, some code editor(I used Visual Studio Code), and some familiarity with command line.

Required Software
  • Git
  • Jekyll (follow the first two instructions)
  • Visual Studio Code
Pre-requisite
  • Github Account
Setup Instructions
  1. Setup the repository you will use for the main website. Follow the user/organization instructions at GitHub Pages
  2. Once you have cloned the main website repository, create and clone another GitHub repository to store the jekyll site code. GitHub Instructions
  3. Create a new Jekyll site in the cloned Jekyll site code repository folder with the command: jekyll new mySite
  4. Update the _config.yml to the information you need.
  5. Update the index.markdown file with whatever you want on your home page.
  6. (Optional) change theme for the site.
  7. commit your changes to git and push to github.
  8. Once you are ready to publish your site, run the command jekyll build to build the site with all the configuration from the _config.yml file.
  9. Run this command to copy _site contents to your xcopy .\_site\* {path_to_repos_from_step_1} /Y/E/H/C/I

Those are basic instructions. Once Github updates to the version of Jekyll you are using you can move the code over an just commit changes to the main repo and GitHub should build your Jekyll site.
